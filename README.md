# 🎤 Debate Rating Service

Сервис для проведения рейтинговых голосований во время дебатов с двумя спикерами.

## 📋 Возможности

- **Real-time обновления** результатов через WebSocket (Socket.io)
- **QR-код для голосования** - один код для всех зрителей
- **Защита от повторных голосов** - один человек может изменить свой голос
- **Автоматический подсчёт процентов** с красивыми анимациями
- **Адаптивный дизайн** для мобильных устройств и больших экранов

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск сервера

```bash
npm start
```

Сервер запустится на порту 3000 (или PORT из переменных окружения).

## 📱 Страницы

| Страница | URL | Описание |
|----------|-----|----------|
| **Голосование** | `http://localhost:3000/` | Страница для голосования зрителей |
| **Рейтинг** | `http://localhost:3000/display` | Экран с рейтингом спикеров |
| **Админка** | `http://localhost:3000/admin` | Панель управления дебатами |

## 📖 Как использовать

### Шаг 1: Запуск дебатов

1. Откройте **Админ-панель**: `http://localhost:3000/admin`
2. Нажмите кнопку **"🚀 Начать дебаты"**
3. Появится QR-код со ссылкой на голосование

### Шаг 2: Голосование

1. Покажите QR-код аудитории
2. Зрители сканируют код и попадают на страницу голосования
3. Выбирают спикера (A или B)
4. Можно изменить голос в любой момент

### Шаг 3: Мониторинг результатов

1. Откройте **Экран рейтинга**: `http://localhost:3000/display`
2. Результаты обновляются в реальном времени
3. Показывается процент и количество голосов

### Шаг 4: Завершение

1. В админ-панели нажмите **"⏹ Завершить дебаты"**
2. Результаты сохраняются до начала новых дебатов

## 🔌 API Endpoints

### POST `/api/debate/start`
Начать новую сессию дебатов.

**Ответ:**
```json
{
  "success": true,
  "sessionId": "debate_1234567890",
  "votingUrl": "http://localhost:3000/"
}
```

### POST `/api/debate/end`
Завершить текущую сессию.

**Ответ:**
```json
{
  "success": true,
  "results": {
    "A": 60,
    "B": 40,
    "totalVotes": 50
  }
}
```

### POST `/api/vote`
Отправить голос.

**Тело запроса:**
```json
{
  "speaker": "A",
  "voterId": "voter_abc123"
}
```

**Ответ:**
```json
{
  "success": true,
  "voterId": "voter_abc123",
  "results": {
    "A": 60,
    "B": 40,
    "totalVotes": 50
  }
}
```

### GET `/api/results`
Получить текущие результаты.

**Ответ:**
```json
{
  "active": true,
  "sessionId": "debate_1234567890",
  "results": {
    "A": 60,
    "B": 40,
    "totalVotes": 50
  }
}
```

### GET `/api/qr?url=...`
Сгенерировать QR-код.

**Параметры:**
- `url` - URL для кодирования

**Ответ:**
```json
{
  "success": true,
  "qrCode": "data:image/png;base64,..."
}
```

## 🔄 WebSocket события

### Событие: `voteUpdate`
Отправляется всем клиентам при изменении результатов.

**Данные:**
```json
{
  "A": 60,
  "B": 40,
  "totalVotes": 50
}
```

## 🔐 Особенности

- **Идентификация избирателей**: Используется localStorage для сохранения ID голосующего
- **Изменение голоса**: Можно переголосовать, выбрав другого спикера
- **In-memory хранение**: Данные хранятся в памяти до перезапуска сервера
- **Масштабируемость**: Поддерживает 30-100 одновременных пользователей

## 🎨 Кастомизация
